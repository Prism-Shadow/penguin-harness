# Web App: isolated in-app previews, steering from the chat input, /model forks, per-turn thinking

## In-app HTML previews use the separate preview origin (#59)

The Files panel's in-app rendered view fed HTML into an iframe via `srcdoc` — a document with no real base URL, so a page's relative subresources (`foo.png`, `app.js`, `style.css`) could never resolve. When preview isolation is available, the rendered view now loads the same separate-origin redirect URL as "open in new page", so relative assets resolve and the page's storage and cookies work — with the in-app iframe sandboxed strictly tighter than the new-tab preview. Without a separate preview origin, the `srcdoc` fallback and its warning semantics remain as before.

## The chat input stays live while a task runs (#63)

The input no longer disables during a run: Enter sends steering — delivered to the model with the next tool result — Stop sits alongside the send button, and the running placeholder explains the delivery. `[user_steering]` blocks render as small user-styled chips inside the tool output view, while the Traces view keeps the raw text.

## /model forks the conversation to another model (#64)

A `/model` slash command in an active idle session opens the model picker; picking a different model forks the conversation onto it and navigates to the new session, posting any text left after the command as its first task — mirroring the `@` handoff flow. The forked session opens under a provenance banner linking back to the source conversation.

## Thinking level is editable mid-session (#64)

The read-only thinking tag in an active session becomes an editable picker — a per-turn override riding on each task, defaulting to "follow agent config" and never writing through to the Agent settings. The draft picker's switch-becomes-default behavior is unchanged.

## Call descriptions in settings and tool cards (#62)

The agent settings Tools tab gains the call-descriptions toggle, and tool-call cards show the model-written call description as the collapsed-header subtitle — the approval row deliberately keeps the real command, since users must approve the actual command rather than the model's summary of it. The new file tools get `file_path`-based previews in the cards.
